# 管理后台

与交易机器人共仓库维护的一套独立管理后台。

## 目录结构

- `admin-api/` — FastAPI 后端，负责认证、总览、账户数据、日志、系统控制
- `admin-web/` — Vue 3 + Vite + Tailwind 前端

## 当前阶段状态

这套后台已经从最初的 MVP 功能页，推进到可持续使用的内部管理台，当前重点不再只是“能看数据”，而是：

- 能看资金、风险、曝险与策略状态
- 能直接做主控启动 / 停止 / 重启
- 能持续自动刷新
- 能在首页直接看到关键异常与风险摘要
- 能通过日志页快速排查问题
- 页面结构已经从单页长卷轴整理为多视图后台

## 当前已完成能力

### 认证与基础框架

- 管理员登录
- 明暗主题切换
- 同仓库部署，前后端独立目录

### 多视图后台结构

前端当前已整理为 4 个主视图：

- **总览**
- **系统控制**
- **交易执行**
- **日志审计**

桌面端带左侧视图切换，小屏也可通过顶部按钮切换。

### 总览页

已支持：

- 账户权益
- 未实现盈亏
- 保证金率
- 持仓名义价值
- 委托名义价值
- 总名义价值
- CTA 状态
- 网格状态
- 风控阻断状态
- 主控进程状态

并且已补两组趋势图：

1. **资金 / 风控主图**
   - 权益
   - 今日盈亏
   - 保证金率
   - 阻断点

2. **曝险 / 阻断走势**
   - 持仓名义价值
   - 委托名义价值
   - 总名义价值
   - 阻断信号

### 风险摘要层

总览页已支持从最近日志中提炼：

- 错误数
- 警告数
- 风控事件数
- 阻断命中数
- 最近重点事件列表

用于让用户无需先进入日志页，也能快速知道系统是否异常。

### 系统控制页

已支持：

- 手动刷新
- 轮询开关
- 启动主控
- 停止主控
- 重启主控
- 主控进程状态查看
- CTA / 网格 / 风控状态摘要
- 控制建议
- 最近告警事件流

并且已加入 **控制动作历史**，记录最近的：

- 手动刷新
- 自动轮询成功 / 失败
- 轮询开关变更
- 启动 / 停止 / 重启
- 取消的控制动作

### 交易执行页

已支持：

- 当前持仓表
- 当前委托表
- 排序
- 粘性表头
- 多空 / 类型 / 盈亏高亮
- 更高密度的后台表格布局

### 日志审计页

已支持：

- 日志表格视图
- 分类筛选
- 模块筛选
- 排序
- 关键词搜索
- 快捷过滤：
  - 全部
  - 异常优先
  - 只看风控
  - 只看 CTA
  - 只看网格
- 当前筛选结果数
- 一键清空筛选

- `/api/auth/me`
- `/api/dashboard/overview`

- daily_pnl
- drawdown_pct
- unrealized_pnl
- margin_ratio_pct
- position_notional
- open_order_notional
- total_notional
- blocked
- blocked_value

## 本地运行

### 启动后端

```bash
cd admin-api
pip install -r requirements.txt
python run.py
```

默认端口：`8008`

默认管理员账号：

- 用户名：`admin`
- 密码：`admin123`

可通过环境变量覆盖：

- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `ADMIN_CONFIG_PATH`
- `ADMIN_LOG_PATH`

### 启动前端

```bash
cd admin-web
npm install
npm run dev
```

可选环境变量：

- `VITE_API_BASE=http://127.0.0.1:8008`
